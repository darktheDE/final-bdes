# Process Log: Scrapy Playwright & 403 Forbidden Fix

## 1. Main Goal
Our primary objective is to create an automated web scraper to extract restaurant information and user reviews from the TripAdvisor website for Ho Chi Minh City. This scraper will feed raw data into the project's MongoDB and HDFS backend for big data processing (MapReduce) and later visualization via Streamlit.

To achieve this, we are utilizing **Scrapy** in combination with **Playwright**. Playwright allows us to render JavaScript-heavy dynamic content and navigate TripAdvisor effectively, overcoming the limitations of standard static HTML crawling.

## 2. Actions Taken
1. **Environment Setup:** Initialized a Python virtual environment (`.venv`) and installed the required packages: `scrapy`, `scrapy-playwright`, and `playwright-stealth`. We also installed the Playwright Chromium browser.
2. **Project Initialization:** Created the Scrapy project `tripadvisor_job` inside the `src/crawler/` directory.
3. **Item Definition:** Mapped out the data fields in `items.py` to match the exact schema defined for our MongoDB database (`_id`, `name`, `rating`, `review_count`, `address`, `district`, `city`, `cuisines`, `price_range`, `reviews`).
4. **Spider Architecture:** Built a `CrawlSpider` to natively support URL rule extraction:
   - A pagination rule matching `-oa<number>-Ho_Chi_Minh_City.html` to navigate through list pages.
   - A review parsing rule matching `Restaurant_Review-g293925-d<id>-Reviews-` to open individual restaurant pages.

## 3. Problem Faced: HTTP 403 Forbidden
During our initial connection test using standard `scrapy-playwright` configurations, the crawler received an immediate **HTTP 403 Forbidden** status code from TripAdvisor, and the spider was unable to extract any HTML links.

## 4. Why It Occurred
TripAdvisor employs aggressive anti-bot protection mechanisms (such as DataDome or PerimeterX). These systems detect automated browsers by analyzing specific footprints. Even though Playwright uses a real Chromium browser, the browser exposes a property called `navigator.webdriver = true` and other automated behavioral quirks. TripAdvisor's firewall flagged our Scrapy spider as an automated bot due to these footprints and immediately blocked the connection.

## 5. Our Solution
To successfully bypass the block, we implemented **Stealth Mode** across the crawler configuration:
1. We modified `settings.py` to include `PLAYWRIGHT_LAUNCH_OPTIONS`.
2. We appended the argument `--disable-blink-features=AutomationControlled`, which explicitly prevents Chromium from announcing itself as an automated browser.
3. We installed `playwright-stealth` alongside setting a genuine `USER_AGENT` string to mask our crawler as a regular human user.
4. We disabled `ROBOTSTXT_OBEY` to ignore TripAdvisor's strict crawler limitations.

By combining these stealth techniques, the anti-bot protection was evaded, and subsequent connection tests successfully yielded **HTTP 200 OK** and extracted 35 new restaurant links from the first page without being blocked.
