🚨
Triệu chứng

MySQL Decimal Type không serialize được

🚨
Triệu chứng
TypeError: Object of type Decimal is not JSON serializable. Script crash khi xuất dữ liệu từ MySQL sang JSON.
🔍
Nguyên nhân
MySQL driver Python trả về kiểu Decimal cho các cột NUMERIC/FLOAT. Python's built-in json.JSONEncoder không biết cách xử lý Decimal — chỉ handle int, float, str cơ bản.
✅ Giải pháp
  • Tạo class MySQLJSONEncoder kế thừa từ json.JSONEncoder
  • Override phương thức default() để kiểm tra kiểu Decimal
  • Convert Decimal → float trước khi serialize
  • Dùng encoder này cho tất cả json.dumps() calls trong script
🎯 Kết quả đạt được
Script chạy hoàn thành, xuất đúng 1,334 restaurants + 44,000 reviews + 666 meals sang JSONL.

MongoDB ObjectId không serialize được

🚨
Triệu chứng
TypeError: Object of type ObjectId is not JSON serializable. Export MongoDB sang JSONL fail tại field _id.
🔍
Nguyên nhân
MongoDB sử dụng BSON ObjectId làm _id — là binary object 12 bytes, không phải string. json.JSONEncoder không xử lý được kiểu dữ liệu này.
✅ Giải pháp
  • Tạo class MongoJSONEncoder với override tương tự
  • Convert ObjectId → str() trước khi serialize
  • Đổi tên field _id → id để Hive không bị lỗi tên field đặc biệt
  • Áp dụng cho tất cả documents trong collection khi export
🎯 Kết quả đạt được
MongoDB export hoàn toàn thành công. _id được rename → id, Hive query không bị conflict field name.

Workflow — Mapper / Combiner / Reducer

Mỗi job theo cùng một pattern: Mapper phát key-value, Combiner tổng hợp cục bộ, Reducer tổng hợp toàn cục.

M
Mapper — Phân tích từng dòng
Đọc từng JSON line từ HDFS. Extract key và value tương ứng. Emit cặp (key, value) cho bước tiếp theo. Bọc trong try/except để bỏ qua JSON lỗi.
Input
JSON line từ HDFS file
Output
Cặp (key, value) trung gian
C
Combiner — Tổng hợp cục bộ
Tổng hợp kết quả trên cùng 1 mapper node trước khi gửi qua mạng. Giảm đáng kể lượng data transfer trong shuffle phase.
Input
Key-value từ mapper cùng node
Output
Key-value đã tổng hợp sơ bộ
R
Reducer — Tổng hợp toàn cục
Nhận tất cả values cho cùng key từ mọi node. Tính kết quả cuối: avg_rating, count, sentiment_score... Xuất ra HDFS output directory.
Input
Key + tất cả values từ mọi mapper
Output
HDFS /data/output/mr_*/part-00000

Malformed JSON Lines làm crash toàn bộ Job

🚨
Triệu chứng
JSONDecodeError làm crash toàn bộ MapReduce job. Một vài dòng trong HDFS file không phải valid JSON, nhưng lỗi này kéo sập cả job đang chạy.
🔍
Nguyên nhân
Dữ liệu từ MongoDB/MySQL export có thể chứa invalid UTF-8 characters, dấu ngoặc kép không escaped, hoặc dòng bị corrupt trong quá trình ghi file.
✅ Giải pháp
  • Bọc toàn bộ json.loads() trong try/except trong mọi mapper
  • Bắt cả JSONDecodeError, ValueError và Exception tổng quát
  • Dòng lỗi được bỏ qua im lặng — job tiếp tục xử lý dòng kế
  • Pattern này được áp dụng nhất quán trong tất cả 8 jobs
🎯 Kết quả đạt được
Job chạy hoàn thành dù có dòng lỗi. Dữ liệu hợp lệ được xử lý đầy đủ, dữ liệu lỗi bị bỏ qua an toàn.

Danh sách 300+ nguyên liệu → regex NFA tạo ra state explosion khi compile naively. Pattern quá phức tạp làm engine backtrack liên tục.
✅ Giải pháp — Tối ưu hóa pattern
  • Sort danh sách ingredients theo độ dài giảm dần — match "beef" trước "be"
  • Escape tất cả ký tự đặc biệt trong tên nguyên liệu
  • Compile 1 pattern duy nhất trong mapper_init() — chạy 1 lần duy nhất
  • Tất cả mapper instances tái sử dụng pattern đã compile, không compile lại
🎯 Kết quả đạt được
Mapper_init chạy nhanh, bộ nhớ ổn định. Job hoàn thành đúng, detect được nguyên liệu phổ biến trong reviews.

Memory Overflow trong Reducer

🚨
Triệu chứng
Container killed due to memory limit exceeded. Shuffle phase bị treo. Job bị YARN terminate trước khi hoàn thành.
🔍
Nguyên nhân
Reducer gọi list(values) → load toàn bộ values của 1 key vào RAM cùng lúc. Key như "Unknown" district có hàng triệu values → tràn bộ nhớ.
✅ Giải pháp — Streaming approach
  • Thay list(values) bằng vòng lặp for val in values — stream từng giá trị
  • Tích lũy total và count incremental, không lưu toàn bộ vào memory
  • Cấu hình mrjob.conf: mapreduce.reduce.memory.mb = 2048
  • Memory usage được bounded constant dù có bao nhiêu values
🎯 Kết quả đạt được
Tất cả 8 jobs chạy hoàn thành trên cluster. Memory usage ổn định dù dataset lớn.

MySQL Connection Fail từ WSL2

🚨
Triệu chứng
Connection refused: 127.0.0.1:3306. Streamlit app chạy trong WSL2 không thể kết nối MySQL đang chạy trên cùng máy.
🔍
Nguyên nhân
WSL2 và Windows là 2 network namespace riêng biệt. "localhost" từ WSL2 không trỏ đến Windows host. MySQL lắng nghe trên Unix socket thay vì TCP port.
✅ Giải pháp
  • Dùng host="127.0.0.1" thay vì "localhost" — chỉ định rõ IPv4
  • Bật use_pure=True → Force Python driver dùng TCP socket, không Unix socket
  • Đổi MySQL password thành "" (empty) — khớp với cấu hình install_infra.sh
  • Đảm bảo MySQL bind-address cho phép connections từ WSL2 subnet
🎯 Kết quả đạt được
Streamlit app kết nối MySQL thành công. Tất cả CRUD operations hoạt động đầy đủ.

HiveServer2 Connection Timeout

🚨
Triệu chứng
Connection refused: localhost:10000. Charts không load được. Hive queries timeout không phản hồi.
🔍
Nguyên nhân
HiveServer2 daemon chưa start, port 10000 bị block, hoặc Hive metastore khởi động chậm. Không có cơ chế fallback → app crash.
✅ Giải pháp — 3-Layer auto-detect
  • get_hive_status() tự probe: thử PyHive TCP trước
  • Nếu fail → thử subprocess hive -S -e
  • Nếu cả 2 fail → dùng mock data pre-computed để app vẫn hiển thị
  • Mỗi chart có mock data riêng — luôn show đúng format
🎯 Kết quả đạt được
Dashboard không bao giờ crash. Hive offline → app tự chuyển sang offline mode, vẫn hiển thị đầy đủ 6 charts.
